# Codebase Bug Scan Plan

## Summary
A comprehensive review of the VisualNote pipeline codebase identified several bugs and code quality issues ranging from validation failures to deprecated API usage and hardcoded paths.

## Identified Bugs & Issues

### 1. Missing `image_query` repair in manifest validation
- **File:** `pipeline/script_gen.py`
- **Lines:** 94-108
- **Issue:** The `_repair_manifest` function attempts to fix common LLM omissions (e.g., `manim_prompt`, `html_content`), but it does not set a default for `image_query`. Since `REQUIRED_SCENE_KEYS` includes `"image_query"`, any LLM response omitting this key will fail validation and crash the pipeline, even after the repair pass.
- **Fix:** Add `scene.setdefault("image_query", None)` inside the scene repair loop.

### 2. Incorrect voice fallback in TTS client
- **File:** `pipeline/clients/http_client.py`
- **Line:** 224
- **Issue:** The `synthesize` method uses `"voice": voice or self.tts_model`. If the `voice` argument is empty or `None`, it falls back to the model name (e.g., `"mimo-v2.5-tts"`), which is not a valid voice identifier and will cause the TTS API to reject the request.
- **Fix:** Change the fallback to a valid default voice name, e.g., `"voice": voice or "Chloe"`.

### 3. Hardcoded absolute path in main script
- **File:** `run_deep_biology.py`
- **Line:** 34
- **Issue:** `PDF_PATH` is hardcoded to `r"d:\tagmango\Module 2 - The Fundamentals of Building Blocks of Life.pdf"`. This will cause the script to fail on any other machine or if the file is moved.
- **Fix:** Make `PDF_PATH` relative to `PROJECT_ROOT` (e.g., `PROJECT_ROOT / "Module 2 - The Fundamentals of Building Blocks of Life.pdf"`) or configurable via CLI argument/environment variable.

### 4. Confusing/redundant `re` import
- **File:** `hyperframes_render.py`
- **Lines:** 591-592
- **Issue:** Inside `_probe_nvenc`, there is `import re as _re`, but the subsequent line uses `re.MULTILINE`. While `re` is imported at the module level (line 30), the local `import re as _re` is redundant and creates inconsistency.
- **Fix:** Remove `import re as _re` and rely on the module-level `import re`.

### 5. Deprecated `datetime.utcnow()` usage
- **File:** `hyperframes_render.py`
- **Line:** 568
- **Issue:** `_dt.datetime.utcnow()` is deprecated in Python 3.12+ and raises warnings.
- **Fix:** Replace with `_dt.datetime.now(_dt.timezone.utc)`.

### 6. Redundant function calls
- **File:** `hyperframes_render.py`
- **Lines:** 431-434
- **Issue:** `_find_visual(scene)` is called twice in the same boolean expression.
- **Fix:** Assign the result to a local variable `visual = _find_visual(scene)` and reuse it.

### 7. Dead code (Minor)
- **File:** `pipeline/tts.py`
- **Line:** 154
- **Issue:** `synthesize_scenes` is defined but never used (the main script loops over scenes and calls `synthesize_scene` individually).
- **Fix:** Remove the unused function to reduce maintenance burden.

## Proposed Implementation Steps
1. Apply the 7 fixes listed above.
2. Run a quick syntax check (`python -m py_compile`) on the modified files to ensure no regressions.
